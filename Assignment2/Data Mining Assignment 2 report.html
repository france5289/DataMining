分類問題設計

因為本身有在研究如何透過數據分析來判斷一個選手的貢獻程度，因此這次作業便打算透過 Decision Tree 與 Softmax regression 來對打者數據的進行分析，判斷打者的打擊能力好壞程度。

資料集說明

透過 player_data_generator.py 生成打者的打擊成績，其包含以下屬性：

  • AB：打席數
  • H：安打數
  • 2B：二壘安打數
  • 3B：三壘安打數
  • HR：全壘打數
  • RBI：打點
  • SB：盜壘數
  • BB：獲得保送次數
  • SO：被三振數
  • HBP：被觸身次數
  • SF：透過犧牲打上壘次數
  • BA：打擊率
    𝐵𝐴=𝐻𝐴𝐵BA=HAB
  • OBP：上壘率
    𝑂𝐵𝑃=𝐻+𝐵𝐵+𝐻𝐵𝑃𝐴𝐵+𝐵𝐵+𝐻𝐵𝑃+𝑆𝐹OBP=H+BB+HBPAB+BB+HBP+SF
  • SLG：長打率
    𝑆𝐿𝐺=𝐻+2×2𝐵+3×3𝐵+4×𝐻𝑅𝐴𝐵SLG=H+2×2B+3×3B+4×HRAB
  • OPS：整體攻擊指數（label value）
    𝑂𝑃𝑆=𝑆𝐿𝐺+𝑂𝐵𝑃OPS=SLG+OBP

打者評價規則

透過OPS我們能大致對打者進行分類，這次作業中為了減少複雜度，我們預計將打者分為三類：

  1. 若打者的OPS高於聯盟中百分之75的選手，則評價為A
  2. 若打者的OPS位於聯盟聯盟與第四分位數之間，則評價為B
  3. 若打者的OPS低於聯盟平均，則評價為C

訓練與測試資料生成

因為我們的在資料生成時是透過OPS來判定一個打者的攻擊能力，因此在訓練模型時，training data與testing data都不應該存在OPS SLG OBP屬性。而同時為了簡化模型避免 Decision Tree 的判斷規則太過複雜，因此我們同時移除BA SB SO RBI屬性。

接著我們將Rank欄位與其他欄位分開，產生輸入資料集X與 label data y。


最後透過scikit-learn中的train_test_split產生 training 與 testing dataset。

Softmax Regression

Softmax Regression 為 Logistic Regression 的衍生模型，其改善了 Logistic Regression 只能分兩類的問題。
當 input instance x 輸入 Softmax Regression model 時，會先計算 x 屬於類別 k 的分數 𝑆𝑘(𝑥)Sk(x)，接著再使用 Softmax function 計算 x 屬於哪一類。

模型參數設定

使用 scikit-learn 中的 LogisticRegressionCV 建立一個 Logistic Regression model 且內建 cross validation 功能。而我們只需要將 multi_class 這個參數設為 multinomial 即為 Softmax Regression model。

參數設定值

  1. cv:10
    進行 10 fold cross validation
  2. multi_class:multinomial
    使用 Softmax function，即為 softmax regression
  3. random_state:42
  4. Cs:5
    regularization scale
  5. scoring:f1_micro
    使用 F1_micro 計算 F1 score

實驗結果

Softmax regression F1 score: 0.9871794871794872

從F1 Score 可以發現 Softmax Regression 表現得十分突出，在這次分三類的問題上表現幾乎完美。然而也許是因為這次的實驗數據設計的較好，使得 label 之間是互斥的，若 label class 之間並不是互斥關係，則 Softmax Regression 可能就沒辦法有這麼漂亮的表現。因為 Softmax Regression 一次只能 predict 一個類別，所以在 label class 之間有互斥關係的情況下，Softmax Regression 才能有好的 performance。
